Skupper Glossary

Virtual Application Network (VAN)

is an application-layer network that enables communications while avoiding the restrictions of the TCP/IP protocols.

Existing applications can run on a VAN without modification. The VAN is a directed graph, although communication is bidirectional.

Site

is a named context for running Deployments

In Kubernetes, the Site name defaults to the namespace name.

Connection

is a channel for communication between two Sites. It is the basis for communication.

A Connection is established using a ConnectionToken.

VanRouter

is the software that runs on each Site that enables connections.

Address

Addresses are created by service bindings to provide targets for messages.

Binding

binds a Deployment or a set of ServiceProcesses to a Address

Service

is user-defined functionality exposed as a network-accessible interface.

In Kubernetes, the Service name defaults to the deployment name and is associated with Deployments. Services can be combined, for example, a product-browse service can be combined with a product-order service.

  • Corresponds to one Kube service name, but not one Kube service.

  • Corresponds to one or more Kube services, but only one Kube service name.

Deployment

is a configuration of the entities that implement a Service in a Site.

In Kubernetes, the Deployment name defaults to the deployment name and is associated with Processes.

ServiceProcess

is a piece of running code that implements the network-accessible interface of its Service.

In Kubernetes, a Process corresponds to a pod.

Channel

is a asymmetric, bidirectional) relationship between two Processes, typically referred to as a client-server relationship.

The Channel endpoints are two Processes with the Channel forming the links in the VAN topology for an application.
